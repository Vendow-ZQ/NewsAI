### 2. 过程中遇到的特别不顺、卡住很久的情况？

#### 卡点一：小审小改无限循环（卡住3小时）

**现象**：
- 小审不断写入Agent协作日志，1小时内写了几百条
- 审改循环永不退出，状态始终为"审改中"

**根因分析**：
1. **字段名错位**：小文/小播只写文档ID字段，不写内容字段，小审读取不到内容
2. **状态检查逻辑错误**：小审检查"审改记录"旧字段判断已审查状态，但该字段已废弃
3. **小改空转**：小改节点没有实际调用EditorAgent，只是返回空结果

**破局方法**：
- **数据流追踪**：打印每个Agent的输入/输出字段，定位数据断层
- **代码审查**：逐行检查reviewer.py的_is_already_reviewed方法，发现字段名错误
- **双写策略**：小文/小播同时写入"内容字段"（供小审查阅）和"文档ID字段"（供链接跳转）

**经验沉淀**：
- Agent间数据契约必须明确定义，字段名变更需同步修改上下游
- 状态机转换必须打印日志，方便追踪状态流转

#### 卡点二：LLM API超时导致流程中断（卡住2小时）

**现象**：
- Doubao API响应时间不稳定，经常超过120秒
- LangGraph一旦超时，整个流程中断，已执行的工作丢失

**根因分析**：
- LLM.invoke()默认超时120秒，Doubao在复杂生成任务上经常超时
- 没有降级方案，超时即失败

**破局方法**：
- **超时调优**：将timeout从120秒调整到300秒
- **Mock Client兜底**：实现MockLLMClient，当真实LLM超时时返回mock数据，确保流程不中断
- **重试机制**：添加@retry装饰器，失败时重试3次

**经验沉淀**：
- 外部API必须假设会失败，设计降级方案
- 开发/测试阶段可用Mock Client，不依赖真实LLM

#### 卡点三：字段类型转换频繁报错（卡住1.5小时）

**现象**：
- 每次写入Bitable都报FieldConvFail错误
- URL、MultiSelect、Datetime三种字段类型轮番报错

**根因分析**：
- 飞书Bitable API对字段格式要求严格
- Python数据类型与飞书字段类型没有自动转换

**破局方法**：
- **单元测试**：每个字段类型单独测试，找出正确格式
- **封装工具函数**：convert_datetime_to_timestamp、format_url等
- **统一入口**：所有Agent通过FeishuStorage写入，统一做类型转换

**经验沉淀**：
- 第三方API的字段类型必须逐一验证，不能假设兼容性
- 数据写入前做schema验证，提前发现格式错误

#### 复用物二：审改循环防无限循环机制

**价值**：确保审改循环最多N轮后退出，防止资源浪费

```python
# 双重保护机制
# 1. 审查侧：检查是否已审查但未修改
def _is_already_reviewed(self, topic):
    review_round = topic.get('审改记录', {}).get('审查次数', 0)
    edit_round = topic.get('审改记录', {}).get('修改次数', 0)
    if review_round > edit_round:
        return True  # 已审查，等待修改
    return False

# 2. 修改侧：无修改则强制通过
def _write_storage(self, ...):
    if not edit_results:
        # 强制通过所有待审选题
        for topic in topics:
            if topic['状态'] == '审改中':
                self.storage.update(..., {'状态': '待发布', '审改状态': '已通过（无修改）'})
```

**复用场景**：
- 任何需要循环审查的工作流
- 人工审核系统（人机协作）
- 自动化质量检查流程

#### 复用物三：飞书Bitable字段类型处理标准

**价值**：避免字段类型转换错误，统一数据格式

```python
class FeishuTypeConverter:
    @staticmethod
    def datetime(dt):
        """Python datetime → 毫秒时间戳(UTC+8)"""
        import pytz
        beijing = pytz.timezone('Asia/Shanghai')
        return int(dt.astimezone(beijing).timestamp() * 1000)
    
    @staticmethod
    def url(url):
        """确保URL包含协议前缀"""
        if not url.startswith(('http://', 'https://')):
            return f"https://{url}"
        return url
    
    @staticmethod
    def multi_select(tags):
        """List → 逗号分隔字符串"""
        if isinstance(tags, list):
            return ", ".join(tags)
        return tags
```

**复用场景**：
- 任何对接飞书Bitable的项目
- 其他Low-code平台的数据对接
