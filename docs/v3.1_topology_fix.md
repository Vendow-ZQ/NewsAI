# v3.1 修复：生产组拓扑改造（小文先跑，小图小播翻译）

## 背景

ZQ 发现一个真正的设计 bug：当前生产组（小文/小图/小播）三人完全并行，
各自独立想象选题内容，产出的三份文档对不上号。

修复方案：把小文从"并行 3 人之一"改为"串行第 1 人"，小图/小播改为
"基于小文长文翻译扩展"。

## 改动清单（6 处改动）

### 改动 1：core/graph/builder.py 改 fan-out 拓扑

旧的边：
```python
workflow.add_edge("小编", "小文")
workflow.add_edge("小编", "小图")     # 删
workflow.add_edge("小编", "小播")     # 删
workflow.add_edge("小文", "production_sync")
workflow.add_edge("小图", "production_sync")
workflow.add_edge("小播", "production_sync")
```

新的边：
```python
workflow.add_edge("小编", "小文")
workflow.add_edge("小文", "小图")     # 新增
workflow.add_edge("小文", "小播")     # 新增
workflow.add_edge("小图", "production_sync")
workflow.add_edge("小播", "production_sync")
# 小文 不再直接进 sync
```

### 改动 2：core/agents/visual_designer.py 重写 prompt

核心改造：从"自己想象 5-8 张图"改为"翻译小文的 [配图N: 描述] 占位"。

关键变化：
- SYSTEM_PROMPT 强调"翻译而非创作"
- user prompt 增加"核心原则：翻译而非创作"规则块
- 输出字段增加"对应小文占位"
- 翻译图数量必须 = 小文占位数量
- 补充图 ≤ 3 张

### 改动 3：core/agents/script_writer.py 重写 prompt

核心改造：从"基于长文做参考的视频脚本"改为"严格基于长文翻译为视频脚本"。

关键变化：
- SYSTEM_PROMPT 强调"翻译长文，不另起炉灶"
- user prompt 增加"核心原则：翻译长文"规则块
- 必须保持选题角度/核心信息点/钩子调性
- 长文章节 → 视频分段
- 长文配图占位 → 视频画面切换点

### 改动 4：core/agents/visual_designer.py 强制读小文长文

- 拓扑保证小文已完成，必须读取小文长文
- 如果文案文档链接不存在，抛 RuntimeError
- 读取带重试（最多3次，间隔2秒）
- 长文内容过短也抛错

### 改动 5：core/agents/script_writer.py 强制读小文长文

- 同小图，强制读取小文长文
- 不存在或失败抛 RuntimeError

### 改动 6：文档更新

- 创建本文档 docs/v3.1_topology_fix.md

## 验证要点

### 拓扑验证
```
小编 → 小文 → [小图 + 小播] → production_sync → 小审
```

### 内容一致性验证（核心）
1. 小文 [配图N] 占位 → 小图详细 prompt 一一对应
2. 小文长文章节 → 小播视频分段一一对应
3. 小文钩子调性 → 小播开场钩子保持一致
