# 8 张业务表 Schema 提案

> 参考 KOC_persona.md 和 Agent_roster.md 的格式，一次性输出初版，等 ZQ 批改。
>
> 业务 ID 格式：`{PREFIX}-{YYYYMMDD}-{NNN}`
>
> 写入策略：实时写入，每个 Agent 干完活立刻写一条记录 + 一条 LOG。

---

## 一、信息源配置（SRC）

### 1.1 表前缀
- 表名：`信息源配置`
- ID 前缀：`SRC`（例：`SRC-001`）

### 1.2 字段定义

| 字段名 | 类型 | 必填 | 备注 |
|---|---|---|---|
| `id` | 文本 | ✅ | 主键，格式 `SRC-{NNN}`（无日期，配置类用短 ID） |
| `源名称` | 文本 | ✅ | 如 "arXiv AI" |
| `源类型` | 单选 | ✅ | API / RSS / 爬虫 / Mock |
| `平台` | 单选 | ✅ | arXiv / HackerNews / GitHub / Reddit / 小红书 / 抖音 / X |
| `API端点` | 文本 | | 接口地址或 RSS 链接 |
| `监控关键词` | 多选 | ✅ | 如 "AI, LLM, Agent" |
| `抓取频率` | 单选 | ✅ | 实时 / 每小时 / 每天 / 手动 |
| `权重` | 数字 | ✅ | 0-1，影响排序 |
| `启用状态` | 单选 | ✅ | 启用 / 停用 / 故障 |
| `最近抓取时间` | 日期时间 | | 上次成功抓取 |
| `平均响应时间` | 数字 | | 毫秒，性能指标 |
| `抓取成功率` | 数字 | | 0-100，百分比 |

### 1.3 使用映射

```
小哨 TrendScout ← 读取全部启用的源，按频率抓取
小数 Analyst   ← 读取响应时间和成功率，评估源质量
```

### 1.4 示例数据

```yaml
id: SRC-001
源名称: arXiv AI 论文
源类型: API
平台: arXiv
API端点: https://export.arxiv.org/api/query?search_query=cat:cs.AI
监控关键词: [AI, LLM, Agent, RAG]
抓取频率: 每小时
权重: 0.95
启用状态: 启用
最近抓取时间: 2026-05-03T14:30:00
平均响应时间: 1200
抓取成功率: 98
```

| `创建时间` | 日期时间 | ✅ | 自动填充 |
| `来源源ID` | 文本 | ✅ | 引用 `信息源配置.id`，如 SRC-001 |
| `平台` | 单选 | ✅ | arXiv / HN / GitHub / Reddit / 小红书 / 抖音 / X |
| `原始链接` | 文本 | ✅ | 原始 URL |
| `标题` | 文本 | ✅ | 原文标题 |
| `摘要` | 长文本 | ✅ | 原文摘要或前 500 字 |
| `作者` | 文本 | | 原作者 |
| `热度值` | 数字 | ✅ | 平台原始热度（分数/点赞/评论数） |

## 四、选题库（TOPIC）⭐ 核心枢纽

### 4.1 表前缀
- 表名：`选题库`
- ID 前缀：`TOPIC`（例：`TOPIC-20260503-001`）

### 4.2 字段定义

| 字段名 | 类型 | 必填 | 备注 |
|---|---|---|---|
| `id` | 文本 | ✅ | 主键，格式 `TOPIC-{YYYYMMDD}-{NNN}` |
| `创建时间` | 日期时间 | ✅ | 自动填充 |
| `更新状态时间` | 日期时间 | ✅ | 状态变更时更新 |
| `选题标题` | 文本 | ✅ | 给内部看的选题名 |
| `选题描述` | 长文本 | ✅ | 核心故事线：讲什么、给谁看、为什么现在讲 |
| `源爆点ID` | 文本 | ✅ | 引用 `爆点洞察.id` |
| `KOC人设对齐度` | 数字 | ✅ | 0-1，小编评估是否符合人设 |
| `推荐平台` | 多选 | ✅ | 公众号 / 小红书 / 抖音 / B站 |
| `内容形态` | 多选 | ✅ | 图文 / 短视频 / 信息图 / 合集 |
| `优先级` | 单选 | ✅ | P0 今日必发 / P1 本周 / P2 储备 |
| `状态` | 单选 | ✅ | 待创作 / 创作中 / 待审核 / 已通过 / 已拒绝 |
| `关联稿件ID` | 文本 | | 创作后回填 `内容稿件.id` |
| `关联分发ID` | 文本 | | 审核通过后回填 `分发计划.id` |
| `小编决策理由` | 长文本 | | 为什么选这个选题 |

### 4.3 状态流转

```
待创作 → 创作中（小编分配给小文）
创作中 → 待审核（小文写完提交）
待审核 → 已通过（小审通过）
待审核 → 已拒绝（小审打回）
已通过 → 待分发（小发接手）
```

### 4.4 使用映射

```
小编 TopicCurator  ← create：爆点采纳后生成选题
小文 ContentWriter ← query：读取"待创作"或"创作中"的选题
小审 Reviewer     ← query：读取"待审核"的选题
小发 Distributor  ← query：读取"已通过"的选题
```

### 4.5 示例数据

```yaml
id: TOPIC-20260503-003
创建时间: 2026-05-03T14:40:00
更新状态时间: 2026-05-03T14:40:00
选题标题: "AI 界的 USB-C 来了？MCP 协议让 Agent 工具互通"
选题描述: |
  故事线：用 USB-C 做类比，讲清楚 MCP 协议解决了什么问题。
  受众：听说过 Agent 但不懂底层协议的产品/运营同学。
  为什么现在：Anthropic 刚发布，窗口期 3-5 天。
源爆点ID: HOOK-20260503-001
KOC人设对齐度: 0.95
推荐平台: [公众号, 小红书, B站]
内容形态: [图文, 信息图]
优先级: P0
状态: 待创作
关联稿件ID: ""
关联分发ID: ""
小编决策理由: "完美契合'非科班大众'定位，USB-C 类比降低理解门槛，情绪钩子强"
```

## 五、内容稿件（DRAFT）⭐ 多平台版本

### 5.1 表前缀
- 表名：`内容稿件`
- ID 前缀：`DRAFT`（例：`DRAFT-20260503-001`）

### 5.2 核心设计：一稿多平台

一条选题对应多篇平台稿件，本表存"母稿"（公众号长文），各平台变体用子字段存储。

### 5.3 字段定义

| 字段名 | 类型 | 必填 | 备注 |
|---|---|---|---|
| `id` | 文本 | ✅ | 主键，格式 `DRAFT-{YYYYMMDD}-{NNN}` |
| `创建时间` | 日期时间 | ✅ | 自动填充 |
| `更新时间` | 日期时间 | ✅ | 每次修改更新 |
| `关联选题ID` | 文本 | ✅ | 引用 `选题库.id` |
| `作者Agent` | 文本 | ✅ | 如 "小文 ContentWriter" |
| **母稿** ||||
| `母稿标题` | 文本 | ✅ | 公众号长文标题 |
| `母稿正文` | 长文本 | ✅ | 完整长文（Markdown） |
| `母稿字数` | 数字 | ✅ | 字数统计 |
| **小红书版本** ||||
| `小红书标题` | 文本 | | |
| `小红书正文` | 长文本 | | |
| `小红书配图说明` | 长文本 | | 小图产出的配图需求 |
| `小红书字数` | 数字 | | |
| **抖音版本** ||||
| `抖音标题` | 文本 | | |
| `抖音正文` | 长文本 | | |
| `抖音脚本` | 长文本 | | 小播产出 |
| `抖音时长` | 文本 | | 如 "45秒" |
| **B站版本** ||||
| `B站标题` | 文本 | | |
| `B站正文` | 长文本 | | |
| `B站脚本` | 长文本 | | 小播产出 |
| `B站时长` | 文本 | | 如 "2分30秒" |
| **视觉产出** ||||
| `封面图prompt` | 长文本 | | 小图产出的文生图 prompt |
| `信息图SVG` | 长文本 | | 小图产出的 SVG 代码 |
| `文字卡片HTML` | 长文本 | | 小图产出的 HTML 模板 |
| **元数据** ||||
| `钩子标题` | 文本 | ✅ | 小析建议的爆点标题 |
| `SEO关键词` | 多选 | | |
| `状态` | 单选 | ✅ | 草稿 / 已完成 / 审核中 / 已通过 / 已拒绝 |
| `审核意见` | 长文本 | | 小审的反馈 |
| `修改次数` | 数字 | ✅ | 0, 1, 2... |

### 5.4 使用映射

```
小文 ContentWriter    ← create：选题采纳后写入母稿 + 各平台版本
小图 VisualDesigner  ← update：配图完成后回填视觉字段
小播 ScriptWriter    ← update：短视频脚本完成后回填抖音/B站脚本字段
小审 Reviewer        ← query + update：审核后更新状态和意见
```

## 八、数据回流（DATA）

### 8.1 表前缀
- 表名：`数据回流`
- ID 前缀：`DATA`（例：`DATA-20260503-001`）

### 8.2 字段定义

| 字段名 | 类型 | 必填 | 备注 |
|---|---|---|---|
| `id` | 文本 | ✅ | 主键，格式 `DATA-{YYYYMMDD}-{NNN}` |
| `创建时间` | 日期时间 | ✅ | 自动填充 |
| `关联分发ID` | 文本 | ✅ | 引用 `分发计划.id` |
| `关联选题ID` | 文本 | ✅ | 引用 `选题库.id` |
| **基础数据** ||||
| `平台` | 单选 | ✅ | 公众号 / 小红书 / 抖音 / B站 |
| `曝光量` | 数字 | ✅ | |
| `阅读量/播放量` | 数字 | ✅ | |
| `互动量` | 数字 | ✅ | 点赞+评论+转发+收藏 |
| `点赞` | 数字 | ✅ | |
| `评论` | 数字 | ✅ | |
| `转发/分享` | 数字 | ✅ | |
| `收藏` | 数字 | ✅ | |
| **率值指标** ||||
| `点击率` | 数字 | | 阅读/曝光 |
| `互动率` | 数字 | | 互动/阅读 |
| `完播率` | 数字 | | 仅视频平台 |
| **内容质量** ||||
| `正向评论占比` | 数字 | | 0-1，情感分析结果 |
| `负向评论关键词` | 多选 | | |
| **复盘** ||||
| `是否达预期` | 单选 | ✅ | 超预期 / 达标 / 未达预期 |

| `选题反馈` | 长文本 | | 对选题库的改进建议 |
