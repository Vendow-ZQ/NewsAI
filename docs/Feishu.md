具体赛题：多维表格上的多智能体虚拟组织
一、竞赛背景与目标
在企业实际运行中，业务的核心并不是单一工具，而是：
多个角色围绕数据协作，通过系统推动流程，并基于数据进行持续决策与优化。
飞书多维表格（Base）作为业务系统构建平台，已经具备：
- 灵活的数据结构与关系建模能力
- 自动化流程与权限控制能力
- 可视化分析与协作能力
与此同时，大语言模型（LLM）与 AI Agent 技术的发展，使得：
- AI 可以承担“员工角色”，执行具体业务任务
- 系统可以从“被动记录”升级为“主动运行”
本次竞赛聚焦于这一前沿方向：
构建一个由 Agent 员工组成的虚拟组织，这些 Agent 作为“数字员工”，通过多维表格 OpenAPI 协同工作，完成系统搭建、业务运行与数据分析。
二、竞赛核心任务
参赛团队需要完成一个虚拟员工系统的构建与运行，其中：
- 多个 AI Agent 扮演不同业务角色（员工）
- 所有业务通过飞书多维表格（Base）进行数据沉淀与状态管理
- Agent 通过飞书和多维表格 OpenAPI 操作多维表格，实现协同工作
- 系统能够持续运行并生成数据
- 最终基于数据完成分析、报告或决策输出
三、系统能力范围与要求
本课题围绕“组织运行系统”，要求参赛作品至少覆盖以下四个核心能力模块：
1. 虚拟员工建模
要求：
- 至少定义 3 个及以上角色（Agent）
- 每个 Agent 对应一个“员工角色”，例如：
  - 运营 / 编辑 / 审核 / 管理者
  - HR / 面试官 / 候选人
  - 用户 / 客服 / 风控
必须包含：
- 角色职责定义
- 输入 / 输出说明
- 角色间协作关系
2. 业务系统构建
Agent 角色基于飞书多维表格构建业务系统：
必须实现：
- 数据表设计（实体建模）
- 字段设计（结构化表达）
- 表间关系（关联记录）
- 状态字段（流程驱动）
技术要求：
- 必须让 Agent 角色通过 多维表格 OpenAPI / SDK/CLI 操作数据
3. 业务运行与协同
构建一个完整业务流程，使 Agent 员工能协同完成任务：
流程要求：
必须形成完整链路：
数据产生 → 状态更新 → Agent处理 → 决策 → 反馈 → 再流转
示例流程：
- 招聘：简历 → 筛选 → 面试 → 评估 → Offer
- 内容：选题 → 生产 → 审核 → 发布 → 反馈
技术实现：
- Agent 员工必须通过调用多维表格 API：
  - 读取数据
  - 更新状态
  - 写入结果
4. 数据分析与报告
系统运行后，需具备数据分析能力，并由 Agent 员工通过多维表格和飞书发送：
必须实现：
- 基于业务数据的自动分析（LLM 或逻辑计算）
- 输出至少一种结果形式：
  - 周报 / 日报
  - 数据洞察
  - 决策建议
5. 参赛约束与公平性
为了保证竞赛的公平性与技术的专注度，所有参赛团队必须在以下框架内角逐：

## 飞书 AI 校园竞赛-产品专场分享会

### 一、产品干货分享（吴星辉）

#### 1. 商业模式画布
- **核心出发点**：思考产品必须从商业模式出发，而非仅关注功能
- **关键要素**：
  - **客户细分**：明确产品面向的用户群体
  - **价值主张**：产品能为用户带来什么价值（避免伪需求）
  - **渠道通路**：产品如何触达客户
  - **关键资源**：做产品的额外价值、自身优势与壁垒
- **成本意识**：建立对金钱的敏感度，准确计算产品成本（开发成本、Token消耗、跑客户成本等）

#### 2. 产品定义方法
- **一句话定义**：用一句话（约15字）概括产品，面向用户、突出差异点、精确表达
- **避免误区**：
  - ❌ 使用高大上但模糊的专业术语（如"工业软件新物种"）
  - ❌ 类比不面向用户
  - ❌ 未突出差异化卖点
- **迭代思维**：产品定义需在不断迭代中反复思考打磨

#### 3. PMF（产品-市场匹配）风险
- **本质认知**：PMF 不仅是产品与市场匹配，而是商业模式画布中各元素的组合，各环节相互耦合且顺序会影响后续决策
- **验证方法**：
  - 产品上线第一天就开始收费——用户愿意掏钱是对价值的真正认可
  - 通过收费做减法，聚焦付费用户需求
  - 建立正反馈，带领团队不断取得小胜仗

#### 4. 项目启动路径
推荐《小而美》的方法论：
- 从社区着手，关注用户问题和现有方案不足
- 产品化前用人工方式做 wait list 和预购，降低 PMF 风险
- 稳扎稳打，用收费验证价值
- **克制营销**：即使融资也要考虑增长核心是靠补贴还是裂变飞轮

#### 5. 给参赛同学的建议
- ✅ 选**小切口**，解决**真实痛点**
- ✅ 有条件时获取**真实用户验证**
- ✅ 能**收到钱**更好（验证价值）
- ❌ 避免资源无限时想解决过多需求

- **体验判断**：发现用户实际行为与预期不一致时，重新打磨体验

#### 2. 协作能力提升
- **敏捷协作**：飞书业务复杂度高、迭代快，需灵活协作、多方及时沟通
- **项目管理**：观察 PMO 同学如何掌握技术业务线和依赖关系，高效对齐卡点

#### 3. 独立负责与 Ownership
- **从 0 到 1 机会**：飞书虽业务成熟，但 AI 创新项目为校招生提供独立负责机会
- **多面手成长**：接触产品功能、策略、运营、设计、宣发等不同工种

#### 4. 校招体验评价
- **优势**：成长性强、薪资有竞争力、AI 生产工具先进、职业背书好、字节文化、办公环境与福利
- **挑战**：实习节奏快、带教风格因人而异
