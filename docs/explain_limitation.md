# 关于飞书 Base 文档的说明

## 当前情况

根据飞书 API 的限制，**无法直接通过 API 在 Base 内部创建独立文档**（像您截图中的"测试"那样在左侧显示）。

## 两种文档类型对比

| 类型 | 位置 | 创建方式 | 示例 |
|------|------|----------|------|
| **Base 内部文档** | 左侧导航栏 | 只能手动创建 | 您截图中的"测试" |
| **Base 表格字段** | 表格单元格内 | 可以通过 API | "帖子内容"字段 |
| **独立云文档** | 我的文档/团队文档 | 可以通过 API | ldx 格式的文档 |

## 当前实现方案

我在"选题库"表中创建了 3 个文档字段：
- `帖子内容` - 小文写入的 4 平台内容
- `视频脚本` - 小播写入的抖音/B站脚本
- `审改记录` - 小审/小改维护的审查记录

在"数据库"表中创建了 1 个文档字段：
- `经验总结` - 小数写入的数据分析

这些字段在表格中显示为可展开的富文本区域。

## 替代方案

如果您确实需要左侧导航栏显示独立的 4 个文档，有两种方法：

### 方案 1：手动创建（推荐）
1. 在飞书 Base 左侧点击"+"
2. 选择"新建文档"
3. 创建 4 个文档："帖子内容"、"视频脚本"、"审改记录"、"经验总结"
4. 将这些文档的链接复制到对应的 Bitable 字段中

### 方案 2：使用独立云文档
- 使用飞书 Docx API 创建独立的 ldx 文档
- 这些文档会出现在"我的文档"或"团队文档"中
- 不在 Base 左侧导航栏显示

## 建议

当前使用 **Bitable 文档字段** 的方案是最实用的：
1. ✅ 可以通过 API 自动写入
2. ✅ 内容随选题/数据记录关联
3. ✅ 支持 Markdown 格式
4. ✅ 可以在表格中直接查看

如果需要左侧显示，建议手动创建 4 个文档，然后修改代码将内容写入这些文档。
