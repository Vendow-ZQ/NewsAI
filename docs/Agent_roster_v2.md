## 一、组织架构

### 1.1 4 部门 + 1 独立

```
NewsAI · 虚拟内容组织架构

┌────────────────────────────────────────────────────┐
│ 总裁办（KOC 本人）                                  │

        │
        ↓ 直接汇报：小编 + 小数
        │
┌────────────────────────────────────────────────────┐

│   ├─ EMP-001 小哨 TrendScout（信息官）              │
└────────────────────────────────────────────────────┘
        ↓
┌────────────────────────────────────────────────────┐
│ Part 2 · 决策组（Curation Dept.）                  │
│   ├─ EMP-002 小编 TopicCurator（选题总编）⭐部门   │
└────────────────────────────────────────────────────┘
        ↓ 派工
┌────────────────────────────────────────────────────┐
│ Part 3 · 生产组（Creation Dept.）                  │
│   ├─ EMP-003 小文 ContentWriter（文字编辑）        │
│   ├─ EMP-004 小图 VisualDesigner（视觉设计师）     │

│   └─ EMP-008 小发 Distributor（分发策略师）        │

        ↓ 发布
┌────────────────────────────────────────────────────┐
│ 独立 · 复盘（Analytics）                            │
│   └─ EMP-009 小数 Analyst（数据分析师）            │
│   ※ 独立追踪发布数据，月度产出经验文档              │

### 1.2 部门 leader 与汇报关系

| 部门 | Leader | 直接汇报对象 |
|---|---|---|
| 信息组 | 小哨（独人组）| 小编 |
| 决策组 | 小编 | KOC（直接对老板负责） |
| 生产组 | 无明确 leader（三人协作） | 小审 |
| 治理组 | 小审 | KOC（小发完成发布后向 KOC 汇报） |
| 独立复盘 | 小数 | KOC（直接对老板负责） |

**对 KOC 直接汇报的 3 位关键员工**：小编（决策入口）、小审（治理把关）、小数（数据复盘）。

## 三、Agent 协作日志格式

### 3.1 字段定义

字段 schema **完全沿用 v1 的 13 字段**，详见 `Agent_roster_v1_deprecated.md` §4.2。

### 3.2 v2 微调

任务类型枚举更新：

```
v1 任务类型（旧）:
  信息采集 / 爆点分析 / 选题生成 / 文案撰写 / 视觉设计 /
  脚本撰写 / 内容审核 / 分发规划 / 数据复盘

v2 任务类型（新）:
  信息采集（小哨）
  选题生成（小编 - 含原小析的爆点拆解）
  文案撰写（小文）
  视觉设计（小图）
  脚本撰写（小播）
  内容审核（小审）

## 六、我替你拍的几个决策

### Flag 1：编号重新排列
**理由**：让顺序与流程一致（信息→决策→生产→治理→复盘），可读性强。
**代价**：与 v1 编号不兼容，但 v1 已归档，无影响。

### Flag 2：小改放在 EMP-007（而非追加到 EMP-010）
**理由**：编号顺序就是流程顺序。小改是治理组的"修改专员"，逻辑上紧跟在小审（EMP-006）后面。

### Flag 3：小编对 KOC 直接汇报（不经过任何中间层）
**理由**：选题决策是 KOC 最关心的事。小编是 KOC 的"军师"，直接对老板负责。

### Flag 4：治理组有 leader（小审），但生产组没有
**理由**：
- 治理组是顺序协作（审→改→发），需要明确的状态把控者
- 生产组是并发协作（文/图/播同时干），三人对等，无需 leader

### Flag 5：小数独立成"复盘"，不放入治理组
**理由**：
- 小数的工作时间不与治理组重叠（在内容发布**之后**才工作）
- 小数对 KOC 直接负责，是"决策伙伴"角色，不是"执行环节"
- 视觉上独立成一个 Part，强调其战略性

### Flag 6：所有 Agent 的"读取/写入"清单显式列出
**理由**：
- 给 Claude Code 写代码时一目了然每个 Agent 该接哪些 Tool
- 评委看到这份清单立刻 get 到"这是个真实分工"
